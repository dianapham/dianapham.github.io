Diana's Take on What to Learn

Sep 17th, 2014

When picking the right language to learn, you’ve really got to think about what you want to do. If you want do web dev, do you want to be back-end, front-end or full stack? If you want to be a mobile dev, what OS do you want to work with: iOS, Android or Windows (cough)? If you wanna make games, do you want to make it for consoles, mobile or PCs? Also, where does money factor in in all of this? Are you switching careers to create things you’re passionate about or to get a salary increase?

Web Dev

Ruby/Ruby on Rails

Ruby on Rails has a huge community that’s rapidly growing (especially in the Seattle area (especially in the Startup community)). Normally, when you hear someone say they know Ruby, they’ll more than likely also know Ruby on Rails. I took the Ruby on Rails bootcamp (equivalent to a dev accelerator now) last year. It was difficult to learn mainly because I almost no experience with programming. However, Rails as a framework takes care of a lot for you and it’s pretty well organized with a bunch of documentation. A lot of people seem to learn Ruby, the language, while they learn Rails, the framework.

Ruby as a language is described as very pretty and easy to use. Semicolons are not required. People create libraries for Ruby and those libraries are called gems (get it? ruby. gem. yeah.). You’ll find yourself using a lot of gems. You can try writing some Ruby here in this short intro tutorial: http://tryruby.org/levels/1/challenges/0

Python/Django

I never learned Django because was sticking with Rails for web dev, but I did learn Python. A lot of people say that Python is more of a general programming language. It’s easy to make prototypes. I learned a bit of Python and liked it. I created some pretty neat games and learned a lot of logic from learning Python. There’s actually a free course going on right now on Coursera for Python if anyone wants to check that out: https://www.coursera.org/course/interactivepython This is one I watched when I learned Python. The instructors are these four old nerdy guys who are just adorably hilarious.

Since I don’t have any experience with Django, here are some resources:

For a general comparison of Django vs Rails, visit this link: http://forums.udacity.com/questions/100012806/why-web-startups-favor-ruby-rails-vs-python-django

For a technical comparison of Django vs Rails, visit this link: https://bernardopires.com/2014/03/rails-vs-django-an-in-depth-technical-comparison/

Node.js

Everyone I’ve meant who knows Node.js pretty much swears by it. The biggest argument is that JavaScript is already everywhere across the web, why not use it throughout the stack? I haven’t had the change to learn it yet, but seriously, apparently people get very hardcore about it once they learn it. Here’s an aptly named article: http://www.toptal.com/nodejs/why-the-hell-would-i-use-node-js

Front-End Dev

Honestly, I think everyone should be a front-end dev. And then if you want to learn back-end too, you’ll be a full stack developer. Anyway, to be a front-end dev, you for surely need to get really good at HTML/CSS/JS. Totally learn jQuery. If you decide you just want to be a web designer, you’ll have an easier time finding a job if you know HTML/CSS. This makes it easier for employers because instead of you just handing off a sketch to a developer, you can actually develop the code yourself. Then you know, the back end developer integrate all the other services and stuff.

Whether you chose to solely do front-end dev or back-end dev, I highly recommend knowing a fair amount of HTML/CSS/JS if you’re getting in web dev. Honestly, I feel like if you want to get a job quick, learning Rails is the way to go right now. From what I’ve gathered in my research, there’s just more companies using Rails than they are companies using Django or Node.js.

Alternatively, if you’re not super worried about what’s the most popular, there are a bunch of other languages and frameworks you could use to make websites, like the ASP.NET framework, AngularJS and others. I’m not a huge fan of sending people to Wikipedia, but this page actually has a really thorough outline of all the web application frameworks sorted by language: http://en.wikipedia.org/wiki/Comparison_of_web_application_frameworks

Mobile Dev

When creating mobile apps, you really have to be aware of how many devices there are in the world. The top OS hitters are iOS, Android and Windows Phone, but within each of those ecosystems, there’s a ton of devices with different resolutions and other hardware customizations. Responsive design is key.

iOS

HOLLA! I really like Obj-C and I’m liking Swift too as I’m learning it. Xcode, an IDE for OS X and iOS, makes it really easy to create projects. The GUI especially makes development much easier (I’m specifically talking about the storyboard). Obj-C’s syntax is enough to drive away a lot of developers, but once you get used to it, it’s not a big deal. Xcode is free to download if you have a Mac computer. It also comes with a simulator so you can test your apps. You do have to purchase a developer license to test on a physical device and submit to the App Store though. Sending out an iOS app to the App Store to be verified can take awhile especially if you have a ton of bugs and they send it back to you, but this is how Apple makes sure they’ve got better quality apps in the App Store. There are a ton of apps already in the App Store, but honestly, making iOS apps is the way you’re going to make more money. The way Apple sets up their user accounts, it’s so much easier for someone to accept purchasing an app. https://developer.apple.com/library/ios/referencelibrary/GettingStarted/RoadMapiOS/

Android

I think Android development was easy for me to pick up because I already learned iOS dev, but really you’re just learning another language’s syntax (Java in this case). The dev kit is free to install, but I found it was a more involved process than downloading Xcode. A good thing is you can develop Android apps on any type of computer (Mac, Windows, Linux). You can customize a lot on Android, which is great, but also can lead to a bunch of bugs and crashes. http://developer.android.com/training/index.html

Windows Phone

The mobile market is saturated with iOS and Android; the Windows Phone really doesn’t compare. But to get things going, Microsoft throws out all sorts of incentives for developers to create apps for Windows Phone. I haven’t keep track of them lately, but they used to do things like give free phones to developers who submit their first app to the store. Or give $250 credit for developers who submit two apps. I dunno. It seems like they’re really grasping. Microsoft just bought Mojang though so I’m sure they’re going to try to get Minecraft on the Windows Phone to boost the popularity of the phone. In any case, while there aren’t many apps in their store, you could say that by submitting your own apps, you’d have more probability of visibility, but at the same time, that’s only visibility from an already low population of Windows Phone users. The language is predominately C#, but I believe you can also use C++. http://msdn.microsoft.com/en-us/library/windows/apps/ff402529(v=vs.105).aspx

Game Dev

if you already know what type of games you want to make and/or what environment, then it’ll be easier to figure out what language you should learn. if you want to make games for Xbox, you’ll want to learn C#. if you want to make web games, learn JavaScript and check out HTML5 Canvas. if you wanna make games for iOS, learn Obj-C or Swift. if you wanna make games for Android, learn Java.

There are also programs that allow cross-platform publishing, but the code won’t be native. Native code means that the programming language used to make the game is the same one the platform is running. (Obj-C for iOS, Java for Android). Games or any app will run faster if you’re using native code. Anyway, if you want to check out making cross-platform games, check out Unity3d which mainly uses C#, Javascript or Boo, then converts to the other platforms. http://unity3d.com/

In Summation

If you’re just starting out and don’t know what you want to do, go to tech events and talk to people about what they do and what they learned to get there. Also, all the basic computer science stuff can be learned with any language. Once you excel at a certain language, you’ll have the fundamental tools to learn another language and see which works best for you and what you want to make. The reason there are a ton of different languages is because some are better for some things than others, but they all basically contain the same concepts.
